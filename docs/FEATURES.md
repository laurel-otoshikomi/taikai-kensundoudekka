## 🏆 大会管理機能

### 1.1 大会作成
**実装ファイル**: `main.js` - `createTournament()`

**機能概要**:
- 新規大会を作成
- 大会名を入力（必須）
- データベースに保存

**使用方法**:
1. 「新規大会を作成」ボタンをクリック
2. 大会名を入力
3. 「作成」をクリック
4. 自動的に作成した大会が開かれる

**データベーステーブル**: `tournaments`
```sql
CREATE TABLE tournaments (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    logo TEXT,  -- base64エンコードされた画像データ
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 1.2 大会選択
**実装ファイル**: `main.js` - `openTournament(tournamentId)`

**機能概要**:
- 既存の大会を選択して開く
- 大会に紐づく参加者・釣果データを読み込み
- ロゴとテーマカラーを適用

**処理フロー**:
```
1. 大会データ取得 (tournaments テーブル)
2. 参加者データ取得 (players テーブル)
3. 釣果データ取得 (catches テーブル)
4. ロゴ表示
5. テーマカラー適用
6. ランキング計算
7. メインページ表示
```

### 1.3 大会削除
**実装ファイル**: `main.js` - `deleteTournament(tournamentId)`

**権限**: 管理者のみ（AUTH_LEVEL === 2）

**機能概要**:
- 大会を削除（関連する参加者・釣果データも削除）
- 確認ダイアログ表示

**注意事項**:
- 削除は取り消せません
- カスケード削除により、関連データも全て削除されます

## 👥 参加者管理機能

### 2.1 参加者登録
**実装ファイル**: `main.js` - `registerPlayer()`

**権限**: スタッフ以上（AUTH_LEVEL >= 1）

**入力項目**:
- **ゼッケン番号**（必須）: 1-999の整数
- **名前**（必須）: 最大50文字
- **所属**（任意）: 最大100文字

**バリデーション**:
- ゼッケン番号の重複チェック
- 空白文字のトリミング
- 数値範囲チェック

**データベーステーブル**: `players`
```sql
CREATE TABLE players (
    id SERIAL PRIMARY KEY,
    tournament_id INTEGER REFERENCES tournaments(id) ON DELETE CASCADE,
    zekken INTEGER NOT NULL,
    name TEXT NOT NULL,
    club TEXT,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(tournament_id, zekken)
);
```

### 2.2 参加者一覧表示
**実装ファイル**: `main.js` - `updateRankings()`

**表示内容**:
- ゼッケン番号
- 名前
- 所属（任意）
- 最大釣果（長さ・重さ）
- ランク（順位）

**ソート順**:
- 長さランキング: 最大の長さの降順
- 重さランキング: 最大の重さの降順

## 🎣 釣果登録機能

### 3.1 釣果登録
**実装ファイル**: `main.js` - `registerCatch()`

**権限**: スタッフ以上（AUTH_LEVEL >= 1）

**入力項目**:
- **ゼッケン番号**（必須）: ドロップダウンから選択
- **長さ**（cm）（任意）: 0.1 - 999.9
- **重さ**（g）（任意）: 1 - 99999

**バリデーション**:
- 長さまたは重さのどちらか必須
- 数値範囲チェック
- 参加者の存在確認

**データベーステーブル**: `catches`
```sql
CREATE TABLE catches (
    id SERIAL PRIMARY KEY,
    tournament_id INTEGER REFERENCES tournaments(id) ON DELETE CASCADE,
    zekken INTEGER NOT NULL,
    length REAL,
    weight INTEGER,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 3.2 釣果編集
**実装ファイル**: `main.js` - `editCatch(id, zekken, length, weight)`

**権限**: スタッフ以上（AUTH_LEVEL >= 1）

**機能概要**:
- 既存の釣果データを編集
- モーダルウィンドウで編集フォーム表示
- 入力値のバリデーション

**処理フロー**:
```
1. 編集ボタンクリック
2. モーダル表示（既存値をプリセット）
3. 値を変更
4. 「更新」ボタンクリック
5. データベース更新
6. ランキング再計算
7. UI更新
```

### 3.3 釣果削除
**実装ファイル**: `main.js` - `deleteCatch(id)`, `showDeleteConfirmDialog(catchId)`

**権限**: スタッフ以上（AUTH_LEVEL >= 1）

**機能概要**:
- 釣果データを削除
- **カスタム削除確認モーダル**を表示（ブラウザネイティブの confirm() ではない）

**カスタムモーダルの特徴**:
- テーマカラーに対応したグラデーション背景
- アニメーション効果（fadeIn, slideIn）
- バックドロップぼかし効果
- 「キャンセル」と「削除する」ボタン
- 背景クリックで閉じる

**処理フロー**:
```
1. 削除ボタンクリック
2. カスタムモーダル表示
3. 「削除する」クリック → データベースから削除
4. 「キャンセル」クリック → モーダルを閉じる
5. 背景クリック → モーダルを閉じる
```

### 3.4 最近の登録履歴
**実装ファイル**: `main.js` - `updateRankings()`

**表示内容**:
- 最新10件の釣果
- **ゼッケン番号と名前を太字で表示**
- 長さ・重さ
- 登録時刻

**表示例**:
```
📋 最近の登録
━━━━━━━━━━━━━━━━━━━━━━
**5番: 山田太郎** - 45.5cm / 1200g
12:34
**12番: 佐藤花子** - 38.2cm / 950g
12:30
...
```

## 📊 ランキング表示機能

### 4.1 長さランキング
**実装ファイル**: `main.js` - `updateRankings()`

**計算方法**:
1. 各参加者の全釣果から最大の長さを取得
2. 降順でソート
3. 順位を付与（同率の場合は同順位）

**表示内容**:
- 順位
- ゼッケン番号
- 名前
- 最大長さ（cm）
- 所属（任意）

### 4.2 重さランキング
**実装ファイル**: `main.js` - `updateRankings()`

**計算方法**:
1. 各参加者の全釣果から最大の重さを取得
2. 降順でソート
3. 順位を付与（同率の場合は同順位）

**表示内容**:
- 順位
- ゼッケン番号
- 名前
- 最大重さ（g）
- 所属（任意）

## 🎖️ トップページ機能

### 5.1 上位3名の大きな表示
**実装ファイル**: `main.js` - `renderTopPage()`

**機能概要**:
- 長さランキング・重さランキングの上位3名を大きく表示
- 表彰式での使用を想定
- メダル絵文字表示（🥇🥈🥉）

**モバイル対応レイアウト**:
```
┌─────────────────────────────────────┐
│ 🥇 1位  | 5番: 山田太郎     | 👆タップで │
│         | 釣り同好会         |  詳細    │
│         | 45.5cm           |          │
└─────────────────────────────────────┘
```

**レイアウトの特徴**:
- 3カラムFlexboxレイアウト
- 左: メダル + 順位（flex-shrink: 0）
- 中央: 名前・所属（flex: 1）
- 右: 「タップで詳細」案内（flex-shrink: 0）

**詳細表示機能**:
- カードをタップすると、その参加者の全釣果を表示
- モーダルウィンドウで表示

### 5.2 カードデザイン
**実装ファイル**: `index.html`, `main.js`

**スタイル**:
- グラデーション背景（primaryColor → secondaryColor）
- 境界線とシャドウ追加（視認性向上）
- 角丸（border-radius: 20px）
- ホバー効果（transform: translateY(-2px)）

### 6.4 ロゴアップロード
**実装ファイル**: `main.js` - `uploadLogo()`

**権限**: 管理者のみ（AUTH_LEVEL === 2）

**機能概要**:
- ロゴ画像をアップロード
- **データベースに保存**（base64エンコード）
- 全ユーザー・全デバイスで表示

**対応フォーマット**:
- JPEG
- PNG
- GIF
- WebP

**表示箇所**:
- ヘッダー（全ページ）
- 大会選択ページ

**注意事項**:
- ファイルサイズは適度に圧縮推奨（1MB以下）
- データベースに保存されるため、localStorage制限なし

## 🔐 権限管理機能

### 7.1 権限レベル
**実装ファイル**: `main.js`

**レベル定義**:
```javascript
// レベル0: ゲスト（閲覧のみ）
// レベル1: スタッフ（釣果の登録・編集・削除）
// レベル2: 管理者（全機能）
```

### 7.2 機能別の権限要件

#### ゲスト（レベル0）
- ✅ 大会選択
- ✅ ランキング閲覧
- ✅ トップページ閲覧
- ✅ 参加者詳細閲覧
- ❌ 大会作成/削除
- ❌ 参加者登録
- ❌ 釣果登録/編集/削除
- ❌ ロゴアップロード
- ❌ テーマカラー変更

#### スタッフ（レベル1）
- ✅ ゲストの全機能
- ✅ 参加者登録
- ✅ 釣果登録
- ✅ **釣果編集**（最近追加）
- ✅ **釣果削除**（最近追加）
- ❌ 大会作成/削除
- ❌ ロゴアップロード
- ❌ テーマカラー変更

#### 管理者（レベル2）
- ✅ スタッフの全機能
- ✅ 大会作成
- ✅ 大会削除
- ✅ ロゴアップロード
- ✅ テーマカラー変更

### 7.3 権限切り替え
**実装ファイル**: `main.js` - `setupAuth()`

**方法**:
ヘッダーの権限レベル表示をクリックすると、デモ用にレベルを切り替え可能。

**本番環境での実装**:
本番環境では、Supabase Authなどの認証システムと連携して、ログインユーザーの権限レベルを管理する必要があります。

## 📱 レスポンシブデザイン

全てのページがモバイルフレンドリーに設計されています。

**対応ブレイクポイント**:
- モバイル: ~767px
- タブレット: 768px~1023px
- デスクトップ: 1024px~

**モバイル最適化**:
- タップしやすいボタンサイズ（最小44x44px）
- 読みやすいフォントサイズ
- スクロール可能なテーブル
- 折り返しを防ぐレイアウト（トップページ）
